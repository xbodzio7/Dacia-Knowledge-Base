# Bigster Catalogue Foundation

Date: 2026-07-21

## Purpose

Create the minimum source-backed Bigster catalogue required by the consumer shortlist and comparison products, using only explicit cells and transmission statements in the official Polish MY26 price list effective 3 July 2026.

## Imported catalogue

- four versions: Essential, Expression, Extreme and Journey;
- fourteen active configurations from non-empty price-matrix cells;
- fourteen gross Polish catalogue prices dated 3 July 2026;
- four source-to-version and fourteen source-to-configuration relationships.

The matrix contains eight manual 4x2 configurations for mild hybrid 140 and mild hybrid-G 140. It also contains three automatic 4x2 Hybrid 155 configurations and three automatic 4x4 Hybrid-G 150 configurations.

## Comparison scopes

Four explicit reporting scopes group the catalogue by powertrain and transmission: mild hybrid 140 4x2 manual, mild hybrid-G 140 4x2 manual, Hybrid 155 4x2 automatic and Hybrid-G 150 4x4 automatic. The scopes contain no invented technical or equipment denominator; they only map the fourteen source-backed configurations into the existing comparison and release products.

## Evidence boundary

The unavailable Essential cells for Hybrid 155 and Hybrid-G 150 are not converted into configurations. Promotional discounts, financing examples, dealer offers and unstated technical values remain excluded. Transmission type is taken from the explicit engine table: six-speed manual, six-speed dual-clutch automatic or automatic Multi-mode as applicable.

## Deferred scope

The Bigster equipment matrix and the configuration mappings for already registered options and packages remain a separate package. This keeps the catalogue foundation independently reviewable and prevents commercial-option logic from being mixed with entity creation.

## Validation

A dedicated regression contract verifies the four versions, fourteen configurations, transmission split, exact selected catalogue prices, omitted unavailable combinations and complete source relationships. The complete repository quality gate is required before merge.
