\# Naming Conventions



\*\*Project:\*\* Dacia Knowledge Base (DKB)



\*\*Version:\*\* 1.0



\*\*Status:\*\* Active



\---



\# Purpose



Defines naming conventions used throughout the project.



\---



\# Documents



Uppercase.



Examples:



\- README.md

\- ROADMAP.md

\- CHANGELOG.md



\---



\# Project Documents



Uppercase with underscores.



Examples:



\- AI\_CONTEXT.md

\- SESSION\_STATE.md

\- AI\_WORKING\_AGREEMENT.md



\---



\# Python



snake\_case



Examples:



\- validate\_data.py

\- build\_report.py



\---



\# CSV



Lowercase.



Plural.



Examples:



\- models.csv

\- engines.csv

\- body\_types.csv



\---



\# JSON



Lowercase.



snake\_case.



\---



\# IDs



Lowercase.



ASCII only.



Stable.



Immutable.



\---



\# Branches



feature/<name>



fix/<name>



docs/<name>



refactor/<name>



\---



\# Commits



Conventional Commits.



Examples:



docs(project):



feat(validation):



fix(import):



refactor(data):



test(validation):
